# Workspace Blueprint

## Rules

- Write in plain, clear language
- Ask clarifying questions before making assumptions
- When you are unsure, say so

## Rule 1 — Always read CONTEXT.md first
Before responding to any request in this workspace, read the `CONTEXT.md` file in the current folder.

## Rule 2 — Every folder has its own CONTEXT.md
The root workspace has a `CONTEXT.md`, and every sub-folder has its own `CONTEXT.md` with specific instructions for that area.

## Rule 3 — Follow the instructions in CONTEXT.md
Whenever you are asked to do something inside this workspace, read the relevant `CONTEXT.md` file first and strictly follow its instructions before taking any action.

## Rule 4 — Folder Creation Protocol
Whenever you are asked to create a new folder in this workspace, you MUST ALWAYS follow this protocol:
1. **Ask for Purpose:** Explicitly ask the user what the mindset or purpose of the new folder is, so you know what to write in its `CONTEXT.md` file.
2. **Create its CONTEXT.md:** Create the new folder and its dedicated `CONTEXT.md` file based on the user's response.
3. **Update Main CONTEXT.md and README.md:** Automatically update the main root `CONTEXT.md` and `README.md` files to add the new folder to the "Folder Map" and "How to navigate" sections.
